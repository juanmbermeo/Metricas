Métricas en el desarrollo del Software

El objetivo primordial de la ingeniería del software es producir un sistema, aplicación o producto de alta calidad. Para lograr este objetivo, los ingenieros de software deben emplear métodos efectivos junto con herramientas modernas dentro del contexto de un proceso maduro de desarrollo del software. Al mismo tiempo, un buen ingeniero del software y buenos administradores de la ingeniería del software deben medir si la alta calidad se va a llevar a cabo. A continuación se verá un conjunto de métricas del software que pueden emplearse a la valoración cuantitativa de la calidad de software

Visión General de los Factores que Afectan a la Calidad

La calidad es un concepto multifacético que puede verse afectado por una amplia gama de factores. Estos factores pueden ser internos o externos a la organización, y pueden ser controlables o no.

Factores Internos:

  • Cultura organizacional: Una cultura que prioriza la calidad crea un ambiente donde los empleados se sienten empoderados para tomar decisiones y tomar medidas para mejorar la calidad.
  • Liderazgo: Los líderes que establecen expectativas claras y brindan apoyo a los empleados son esenciales para una cultura de calidad.
  • Procesos: Los procesos bien definidos y documentados ayudan a garantizar la consistencia y la calidad.
  • Recursos humanos: Los empleados con las habilidades y la experiencia adecuadas son esenciales para producir productos y servicios de alta calidad.
  • Tecnología: La tecnología adecuada puede ayudar a mejorar la eficiencia y la calidad.
  • Infraestructura: Una infraestructura adecuada, como instalaciones y equipos, es necesaria para producir productos y servicios de alta calidad.

Factores Externos:

  • Clientes: Las expectativas de los clientes son un factor importante a la hora de determinar la calidad.
  • Competencia: La competencia puede presionar a las organizaciones para que mejoren su calidad.
  • Regulaciones: Las regulaciones gubernamentales pueden establecer requisitos de calidad específicos.
  • Factores económicos: Los factores económicos, como la inflación y las recesiones, pueden afectar la capacidad de las organizaciones para invertir en calidad.
  • Factores económicos:∑ Los factores sociales, como los cambios en los valores y las actitudes, pueden afectar las expectativas de los clientes.

Medida de la calidad

Las medidas de calidad son herramientas cuantitativas que se utilizan para evaluar diferentes aspectos de un producto, servicio o proceso. Su objetivo es determinar el grado en que se cumplen los requisitos de calidad establecidos y si el producto o servicio es satisfactorio para el cliente.

Medidas de fiabilidad y de disponibilidad.

La fiabilidad se define como la probabilidad de que un sistema funcione sin fallos durante un período de tiempo determinado. Se suele medir como el tiempo medio entre fallos (MTBF), que es el tiempo promedio que un sistema funciona antes de fallar.

La disponibilidad se define como la probabilidad de que un sistema esté operativo en un momento dado. Se suele medir como el porcentaje de tiempo disponible (PTA), que es el porcentaje de tiempo que un sistema está operativo durante un período de tiempo determinado.

Eficacia de la Eliminación de Defectos

La Eficacia de la Eliminación de Defectos (EED) es una métrica fundamental para evaluar la eficiencia de un proceso de desarrollo de software en la identificación y corrección de errores. Se calcula como el porcentaje de defectos encontrados en una fase del proceso que se corrigen antes de pasar a la siguiente.

Factores de Calidad de McCall

El Modelo de Calidad de McCall es un marco de trabajo que define un conjunto de factores para evaluar la calidad del software desde la perspectiva del usuario.

  1. Factores de Operación:
    • Corrección:El software cumple con las especificaciones y funciona sin errores.
    • Eficiencia:El software utiliza los recursos del sistema de manera eficiente.
    • Integridad:El software es capaz de manejar situaciones de error y protegerse de accesos no autorizados.
    • Usabilidad:El software es fácil de aprender y usar.
    • Mantenibilidad:El software es fácil de modificar y corregir.
    • Flexibilidad:El software es adaptable a nuevos entornos y necesidades.
  2. Factores de Revisión:
    • Capacidad de prueba:El software es fácil de probar y verificar.
    • Analizabilidad:El software es fácil de entender y analizar.
    • Modificabilidad:El software es fácil de modificar y adaptar.
    • Portabilidad:El software es fácil de portar a diferentes plataformas.
    • Reusabilidad:Los componentes del software pueden ser reutilizados en otros proyectos.
  3. Factores de Transición:
    • Instalabilidad:El software es fácil de instalar y configurar.
    • Compatibilidad:El software es compatible con otros sistemas y software.
    • Interoperabilidad:El software puede trabajar e interactuar con otros sistemas.
    • Seguridad:El software protege la información y los datos del usuario.
